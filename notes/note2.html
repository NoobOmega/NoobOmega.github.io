这次我们来聊一下面向过程程序设计。上一篇我们对程序设计进行了概述，首先明确输入输出，思考大概的算法，设计对应的数据类型和结构，在在数据结构的基础上进行算法设计和构建。
这次我们具体来聊一聊算法设计中的面向过程编程。
                      
在日常的算法设计中，（例如cg或者洛谷）如果你使用的是面向过程的高级语言（例如c语言），完全可以把代码用函数分割成一个又一个模块。
如此，便于修改和纠错，只需要在各一函数设置断点，就能精确定位至代码段，且过程更加清晰。

可以先在代码中写出你“理想的函数”，然后再跳到文件开头进行函数定义。例如，我想要相加1，2.
int main（）{
  int a=1,b=2;
  int c;//定义数据“容器”
  c=add(a,b)//直接写出“理想函数”
  printf("%d",c);//输出，这里也是一个函数。
  return 0;
}
此时，再跳转到开头对add进行函数定义，你会发现思路更加清晰和明了。
